# UI 実装ルール

## 基本姿勢

- 本文を主役にする
- UI は補助に徹する
- 追加 UI は「無いと困る」場合に限る

## 常設 UI の原則

- サイドバーは原則禁止
- 常設ツールバーは原則禁止
- 本文表示に不要なヘッダー、フッター、メタ情報バーは置かない
- 追加 UI が必要な場合は、本文の可読性を損なわない最小導線にする

## Settings の扱い

- Settings は macOS 標準のアプリメニューから開く
- 現在の明示的な許可項目は `Theme (Light / Dark)` と `Text scale` とする
- 既定値は `Theme = Light` とする
- 設定項目を増やす場合は「本文体験を壊さず、かつ常設 UI で代替できない」ことを説明できなければならない

## Find in Page の扱い

- 本文検索は `⌘F` を基本導線とする
- 検索 UI は一時的な最小検索バーとして出し、常設しない
- 検索結果件数と前後移動は許可する
- 本文より強く主張する検索 UI は避ける

## 見た目の原則

- 余白は十分に取り、窮屈さを避ける
- 情報階層はタイポグラフィと spacing で作る
- 線、影、色は最小限に使う
- コードブロック、表、引用、見出しは Markdown の意味構造が一目で分かるようにする

## モーションの原則

- 状態変化の理解を助ける短いモーションは許可する
- 過剰な spring、長い遅延、派手なトランジションは禁止
- モーションがなくても成立する UI を優先する

## 禁止事項

- 流行表現のためだけの glass / blur / gradient の多用
- 操作頻度の低い機能の常設表示
- 装飾のためのアイコン追加
- 本文より目立つコントロール
