# 文書運用ルール

## 基本方針

- `AGENTS.md` は入口であり、詳細の保管庫ではない
- `DESIGN.md` は理念文書であり、仕様書ではない
- 具体ルールは `docs/rules/` に集約する

## 更新権限

- `docs/rules/` は必要に応じて自動更新してよい
- `DESIGN.md` は変更前にユーザー確認を取る
- repo の思想を変える更新は、コード変更と別に意図を説明できること

## 分割ルール

- 1 つの文書が長くなりすぎたら責務ごとに分割する
- 同じ内容を `AGENTS.md` / `DESIGN.md` / `docs/rules/` に重複記載しない
- 要約は `AGENTS.md`、原文は `docs/rules/` に置く

## README との関係

- README は公開リポジトリ向けの英語ドキュメントとする
- 内部的な設計思想や運用ルールは日本語 docs に置く
- README に内部 docs の存在を案内してよいが、README を運用ルールの正本にしない
