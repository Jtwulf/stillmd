# テスト・確認ルール

## 基本原則

- 新しいロジックには対応するテストを追加する
- docs のみの変更でも、影響範囲に応じて既存テストやビルドの健全性を確認する
- UI は自動テストだけでなく、目視で質を確認する

## 最低限の確認

- `swift build`
- `swift test`

## UI 変更時の目視確認

- 本文が主役になっているか
- light / dark の両方で破綻していないか
- コードブロック、表、引用、リストが読みやすいか
- 不要な UI が増えていないか
- `⌘F` の検索バーが必要時だけ表示されるか
- Theme / Text scale が preview に即時反映されるか
- モーションが静かで短く、邪魔になっていないか

## 性能変更時の確認

- release `.app` のサイズ確認
- 長い Markdown ファイルでの再読み込みとスクロール確認
- 起動直後の体感悪化がないか

## docs 変更時の確認

- `AGENTS.md` が短い入口として保たれているか
- `DESIGN.md` が理念中心に留まっているか
- 詳細ルールが `docs/rules/` に逃がされているか
