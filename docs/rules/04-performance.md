# 軽量性・性能ルール

## 軽量性の定義

stillmd における軽量性は、少なくとも以下を意味する。

- 起動が速い
- メモリ使用量が抑えられている
- 長文 Markdown でも破綻しにくい
- アプリサイズが小さい

## 目標

- release `.app` のサイズは 20MB 以下を強く意識する
- 見た目のためだけに依存や資産を増やさない
- 重いライブラリ導入は、代替案より明確な価値がある場合に限る

## 実装原則

- まずは OS 標準機能と既存構成を活かす
- 常駐する状態やキャッシュは必要最小限にする
- 再描画や監視処理は短い周期で無駄に回し続けない
- UI の豪華さより応答性を優先する

## 変更時の確認観点

- 起動体験が遅くなっていないか
- 長文ファイルのスクロールや再読み込みで体感が悪化していないか
- バンドルサイズが不必要に増えていないか
- 新規依存が stillmd の思想に見合うか
