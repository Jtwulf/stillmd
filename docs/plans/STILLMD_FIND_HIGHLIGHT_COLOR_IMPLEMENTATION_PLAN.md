# stillmd: Command-F 検索ハイライト色 調整 実装計画書

## 位置づけ

- 関連計画: `docs/plans/STILLMD_FIND_PREVIEW_POLISH_IMPLEMENTATION_PLAN.md`
- 本書の対象は `preview.css` の検索ハイライト色のみ
- 検索バーのレイアウト、余白、影、safeAreaInset の構造には触れない
- 目的は「活性時の色をもっと鮮やかにし、検索中であることを直感しやすくする」こと

## 現状の色

`stillmd/stillmd/Resources/preview.css` では、検索ハイライトに次の変数を使っている。

| テーマ | 通常マッチ | 活性マッチ |
|--------|------------|------------|
| Light | `rgba(255, 214, 92, 0.56)` | `rgba(255, 184, 0, 0.78)` |
| Dark | `rgba(255, 214, 92, 0.36)` | `rgba(255, 184, 0, 0.58)` |

見た目としては、やや赤みのある金色寄りで、機能は十分だが「活性中」の強さが少し弱い。

## 新しいカラー案

### 推奨案

同じ黄系統を保ちながら、赤みを少し抑えつつ、黄緑には寄せすぎず、より純粋な黄色寄りに寄せる。

| テーマ | 通常マッチ | 活性マッチ |
|--------|------------|------------|
| Light | `rgba(255, 236, 138, 0.42)` | `rgba(255, 223, 36, 0.92)` |
| Dark | `rgba(255, 224, 120, 0.28)` | `rgba(255, 210, 32, 0.74)` |

### 意図

- 通常マッチは「見つけた場所」を示すだけに留める
- 活性マッチは通常マッチよりかなり強くして、現在位置がすぐ分かるようにする
- 黄色の軸は維持し、黄緑には逃がしすぎない
- inactive はかなり控えめに、active は一目で分かる差を付ける
- どちらのテーマでも、本文の可読性を壊さない範囲に抑える
- 既存の静かなトーンは維持し、過度に蛍光色にはしない

## 実装方針

1. `preview.css` の `:root` と `[data-theme="dark"]` にある `--find-match-bg` / `--find-match-active-bg` を更新する
2. `MarkdownWebView.swift` や `HTMLTemplate.swift` のロジックは変更しない
3. `mark.stillmd-find-match` と `mark[data-find-active="true"]` の区別は維持する

## 検証項目

- light / dark それぞれで、通常マッチと活性マッチの差が一目で分かること
- 本文、見出し、コードブロックの上でもハイライトが過度に目立ちすぎないこと
- 長文検索時に、連続するハイライトが眩しすぎたり読みにくくなったりしないこと
- 既存の検索挙動、スクロール位置、テーマ切り替えに影響がないこと

## 完了条件

- `preview.css` の検索ハイライト色が更新されている
- light / dark の両方で視認性と静けさのバランスが取れている
- 既存の `⌘F` 動作や本文レンダリングに副作用がない
