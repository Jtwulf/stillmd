# stillmd 起動時ウィンドウサイズ固定 実装計画書

## 概要

stillmd は preview-only / ミニマル / 静けさを重視するアプリであり、起動直後のウィンドウサイズもその思想に沿っている必要がある。
現状は初回ウィンドウがやや大きく、空状態での余白が広すぎるため、本文を読む前の印象として少し間延びして見える。

今回の目的は、stillmd の起動時ウィンドウサイズを一貫した値に揃え、現状より小さく静かな初期表示へ調整することにある。

この計画書では、まず「毎回同じ初期サイズで開く」ことを主スコープとする。
必要であれば後続で「ユーザーによるリサイズ自体を制限する」案も扱えるが、それは別判断として分離する。

---

## 1. キャッチアップ結果

### 1.1 現状の実装

- `MarkdownPreviewer/App/MarkdownPreviewerApp.swift` の `WindowGroup` に `.defaultSize(width: 860, height: 700)` がある
- 同じ箇所で `RootView` に `.frame(minWidth: 600, minHeight: 400)` が指定されている
- つまり現状の起動サイズはコード上 **860x700 pt**、最小サイズは **600x400 pt**
- ウィンドウサイズを保存・復元する独自設定や永続化処理は現状見当たらない
- 初期表示が空状態でも、本文表示でも、同じ WindowGroup 設定が使われる
- ただし macOS / SwiftUI の scene restoration により、保存済みウィンドウサイズが `defaultSize` より優先される可能性がある

### 1.2 スクリーンショットから分かること

- 添付画像自体のピクセルサイズは **972x1332 px**
- ただしこれはスクリーンショット画像全体のサイズであり、macOS の表示倍率もあるため、そのままアプリの論理サイズとは一致しない
- 実装上の正本はスクリーンショットではなく `.defaultSize(860, 700)` と見るべき

### 1.3 思想との整合

- `AGENTS.md` / `DESIGN.md` / `docs/rules/02-ui-implementation.md` から、stillmd は本文より強い UI を避けるべき
- 空状態で大きすぎるウィンドウは、本文が存在しない段階で余白だけが強く見えやすい
- したがって、初期サイズを少し絞る方向はプロダクト思想と整合する可能性が高い

---

## 2. 要件整理

### 2.1 主要件

- stillmd 起動時のウィンドウサイズを、毎回同じ値で開くようにする
- 現状の `860x700` より小さい初期サイズへ調整する
- 空状態でも窮屈に見えず、Markdown 表示時にも最低限の読みやすさを確保する
- 多機能な設定 UI は追加しない

### 2.2 非機能要件

- stillmd のミニマル思想を壊さない
- macOS 標準のウィンドウ操作感を不必要に損なわない
- 既存の複数ウィンドウ挙動、ファイルオープン、Dock drop、Finder 連携に影響を与えない
- 既存の build / test を壊さない

### 2.3 今回の前提

- 本計画では「起動時の初期サイズ固定」を主対象とする
- つまり、**毎回同じサイズで開く**ことを優先し、**ユーザーの手動リサイズ可否**は初期実装では変えない前提で進める
- もし「絶対にサイズ変更できないウィンドウ」にしたい場合は、別途その意図を確認したうえで追加実装とする

---

## 3. 推奨サイズ案

### 3.1 現状サイズ

- 現状: **860x700 pt**
- 最小制約: **600x400 pt**

### 3.2 候補比較

1. **760x560 pt**
   - 現状より少し小さい
   - 安全だが、まだやや大きめ

2. **720x520 pt**
   - ミニマルさと実用性のバランスが良い
   - 空状態の間延び感をかなり減らせる
   - 見出し・本文・コードブロックの閲覧幅もまだ確保しやすい

3. **680x480 pt**
   - よりコンパクト
   - 空状態は美しい可能性がある
   - ただしコードブロックや表を含む Markdown では窮屈になりやすい

### 3.3 推奨

結論から言うと、第一候補は **720x520 pt**。

理由は次の通り。

- 現状より十分に小さい
- それでも `600x400` に近すぎず、初回閲覧の読みやすさを残せる
- 空状態と本文表示のどちらにも極端に寄りすぎない
- stillmd の「静かな最小性」に寄せやすい

## 5. 実装メモ

- 最小変更で済ませるなら、主変更点は `MarkdownPreviewer/App/MarkdownPreviewerApp.swift` のみで足りる可能性が高い
- 将来的にサイズを定数化する場合も、設定画面や永続化はまだ不要
- stillmd の思想上、「設定を増やしてユーザーに選ばせる」より「静かな妥当値を最初から与える」ほうが自然
- 実際には `defaultSize` だけでは保存済み復元値に負ける場合があるため、`restorationBehavior(.disabled)` と `LaunchWindowSizer` による初回適用を併用した

---

## 6. 想定リスク

1. サイズを小さくしすぎると、コードブロック・表・画像で窮屈になる
2. 「固定」を非リサイズと解釈すると、macOS 標準挙動を崩す可能性がある
3. 空状態にはちょうどよくても、実ファイル閲覧時には狭く感じる可能性がある

このため、初手は `720x520 pt` 程度の穏当な縮小から入るのが安全。
