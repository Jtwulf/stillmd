The patch introduces a core regression where in-app open workflows never work because `openWindowAction` is never initialized. It also has additional functional gaps around local Markdown link handling and watcher recovery, so the change is not correct as submitted.

Full review comments:

- [P1] Mount the `openWindow` injector before using `WindowManager` — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/App/MarkdownPreviewerApp.swift:8-13
  `WindowManager.openFile(_:)` can only open a new preview if `openWindowAction` has been filled from the SwiftUI environment, but this scene never mounts `OpenWindowInjector` anywhere. In production that means `openWindowAction` stays `nil` for the entire app lifetime, so both File ▸ Open and dropping another Markdown file onto an existing window silently do nothing.

- [P2] Open local Markdown links inside the previewer instead of via LaunchServices — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Views/MarkdownWebView.swift:95-102
  This `file:` branch always falls through to `NSWorkspace.shared.open(url)`, even when the target is another Markdown document. As a result, clicking `[other.md](other.md)` only works if StillMD is already the system default handler for `.md`; otherwise the link opens in some other app, which breaks in-app cross-document navigation despite the comment above promising a new preview window.

- [P2] Keep watching after a delayed delete/recreate save — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Services/FileWatcher.swift:83-95
  Once these retries are exhausted, the watcher only emits `.deleted` and never reattaches to the path. Because the dispatch source is still bound to the old vnode, any editor or generator that recreates the file more than ~1.2s later leaves auto-reload permanently broken until the user closes and reopens the preview.
