The new previewer has multiple functional regressions in core workflows: opening files from inside the app is currently broken, local Markdown links are inert, syntax highlighting is not actually visible, and file watching can stop permanently after slower delete/recreate saves. Those issues mean the patch is not ready to be considered correct.

Full review comments:

- [P1] Inject the openWindow action before calling openFile — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/App/MarkdownPreviewerApp.swift:8-10
  When the app starts without an already-open document, this scene never stores `@Environment(\.openWindow)` into `WindowManager`, because `OpenWindowInjector` is defined but never used here. In that state `WindowManager.openFile` hits `guard let action = openWindowAction else { return }`, so File > Open and drag-dropping another file from an existing window both become no-ops unless the app was launched by an external document event.

- [P2] Open relative file links instead of cancelling them — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Views/MarkdownWebView.swift:93-97
  Any Markdown link that resolves to a local `file:` URL is cancelled here, while the JS click handler only forwards `http/https`, so links like `[other](other.md)` or `[image](./notes/readme.md)` are dead. Because `loadHTMLString` already uses the document directory as `baseURL`, this breaks the advertised relative-link support for local Markdown cross-links.

- [P2] Add highlight.js token styles to the bundled CSS — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Resources/preview.css:92-105
  These rules style the code block container, but the bundle never defines any `.hljs` / `.hljs-*` selectors. `hljs.highlightAll()` will still wrap tokens in span classes, yet without a theme those spans inherit the normal code color, so fenced code blocks remain visually monochrome in both light and dark mode despite claiming syntax highlighting support.

- [P2] Restart the watcher after slow delete/recreate saves — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Services/FileWatcher.swift:79-87
  If an editor saves by renaming the file away and recreating it more than 200ms later, this branch only emits `.deleted` and keeps the dispatch source attached to the old vnode. After that, recreating the file at the same path produces no further events, so auto-reload stays broken until the user closes and reopens the preview.
