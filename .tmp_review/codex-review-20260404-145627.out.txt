The patch introduces a broken cold-start open path, unsafe HTML rendering in the preview web view, and incorrect window focusing for files that share a basename. Those issues affect core app behavior and security, so the patch should not be considered correct.

Full review comments:

- [P1] Don't mark files as open before creating the first window — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Services/WindowManager.swift:18-19
  Because `openWindowAction` is only populated from `PreviewView.onAppear`, it is `nil` on a fresh launch before any preview scene exists. In that state `showOpenPanel()` reaches this branch, inserts the URL into `openFiles`, and then does nothing, so `⌘O`/Dock-open from a cold start opens no window; retrying the same file also fails because the URL is now treated as already open.

- [P1] Sanitize marked output before assigning to innerHTML — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Services/HTMLTemplate.swift:42-42
  When the opened markdown comes from an untrusted source and contains raw HTML like `<img onerror=...>` or a `javascript:` link, `marked.parse` emits it verbatim and assigning the result to `innerHTML` runs that code inside the `WKWebView`. Because the page also exposes native message handlers, simply previewing a malicious `.md` file becomes script execution in the app context.

- [P2] Identify existing windows by full URL instead of title — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Services/WindowManager.swift:43-45
  When two different files share the same basename, reopening one of them hits the duplicate path and this matcher can focus the wrong window because it only compares `window.title`. For example, with `docs/README.md` and `src/README.md` open, reopening the second file can bring the first window to the front even though duplicate detection is based on the full standardized URL.

- [P2] Escape closing script tags in embedded markdown — /Users/justinwulf/Documents/justin/work/querylift/service/stillmd/MarkdownPreviewer/Services/HTMLTemplate.swift:41-42
  This interpolates raw markdown into a `<script>` element without escaping `</script>`. Any document that contains HTML examples or fenced code with that literal sequence will terminate the script tag during HTML parsing, which leaves the preview blank or partially rendered on the initial load.
